Location Data Search

About

What is the goal?

  • People can search for 'electronics store in Brisbane CBD' and find what they're looking for but searching for 'Accident Data Brisbane CBD' returns semi-relevant or irrelevant data.
  • You can search for the data in the 'Input Data' search box andthe location in the 'Location Search' box. This will return all the data it has found.

    • What did you use?

  • Plain NodeJS was used as the server.
  • HTML, CSS & JavaScript was used to develop the client.
  • OpenStreetMaps was used with Leaflet to display the location data.

    • What were the roles?

  • Matthew: Communicating with the APIs and the search.
  • Reuben: Displays the data in a meaningful format.

    • How does it work?

  • 'Data Search' searches the description of the data for matching values.
  • 'Location Search' searches the data's GPS co-ords for if it is within the range.
  • Once the data found it is returned as JSON and is filtered through the output methods.

    • What were some of the difficulties?

  • Getting all the names of the same type of data, e.g Longitude/Long & Latitude/Lat.
  • Interacting with the API.

    • What still needs to be done?

  • Designing the UI.
  • Adding a way to add datasets.
  • More search filters.
